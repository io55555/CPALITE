# Grok Manager v1.3.5

## 变更

### 全量测活同步策略（默认更省隔离）
参数里新增 **同步隔离策略**：

| 策略 | 行为 |
|------|------|
| **仅候选（默认）** | 只把 DELETE_CANDIDATE（默认 401/402/403）写入隔离；**不**把整库 429 灌进黑名单 |
| **全部失败** | 401/402/403/429 都同步（旧行为） |
| **不同步** | 测活只出报告，不改隔离 |

- 测活过程中实时写入隔离也遵循同一策略  
- 手动「同步隔离」同样尊重该策略  
- 选择会记在浏览器 localStorage  

### 继承 1.3.4
- 处理筛选 / 进度条 ETA / auth.list 缓存
